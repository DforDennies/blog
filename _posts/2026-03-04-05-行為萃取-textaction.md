品牌搜尋「運動」，該推薦健身教練還是運動品牌代言人？

兩類 KOL 都跟「運動」有關，但品牌要的完全不同。找人教觀眾做深蹲，推薦健身教練。找人穿球鞋拍帥照，推薦運動品牌代言人。傳統搜尋引擎只看貼文裡有沒有「運動」這個詞，分不出這兩種意圖。

TextAction 就是來解決這個問題的。它從 KOL 的貼文文本中萃取行為意圖，然後把這個行為維度加入搜尋和推薦系統。搜「運動」不只看主題標籤，還看 KOL 在運動這個主題上的行為模式 — 是在「教學」還是在「推薦」？

## 六大行為：一套分類框架

TextAction 定義了六種行為，每種對應不同的意圖強度：

**宣傳**：最中性的行為，純粹傳達資訊。KOL 告訴你某個品牌出了新品，沒有特別推或不推。

**分享**：帶有個人色彩，後面接的是經驗、心得。「我用了這個面膜一個月」這種。

**表達**：捕捉的是情緒。「今天超開心」「這次合作真的很累」，不見得有產品相關資訊。

**推薦**：push 意味最濃的行為，直接鼓勵或呼籲行動。「大家快去買」「這個真的值得入手」。對品牌來說，這類 KOL 的導購能力最直接。

**開箱**：新產品展示，打開包裝的過程。體驗式的內容，重點在第一印象。

**教學**：指導特定任務的步驟。「三步驟教你畫眼線」「居家訓練菜單」。知識型內容，粉絲黏著度通常高。

這六個類別是由 DS/DA 團隊根據社群媒體的內容特性定義的。我認為這套分類的價值在於它把「主題」和「行為」拆開了 — 同樣是美妝主題，「推薦」和「教學」代表完全不同的 KOL 風格和商業價值。

## 模型：seq2seq 的結構化輸出


v1.0 的模型選擇是 asym-t5-small，一個 seq2seq 架構。輸入是貼文文本，輸出是結構化的行為標籤：

```
輸入: "讓自己面對危機時，都能處變不驚。"
輸出: /分享/心得/面對危機
```

這個輸出格式 `/行為/主題/對象` 是刻意設計的。用 `/` 分隔讓模型的輸出天然就是結構化的，不需要額外的 parsing 邏輯。行為是六大類之一，主題和對象是模型從文本中自由生成的。

訓練資料裡有 20% 的 label 是空的，因為不是每篇貼文都有明確行為。一張單純的風景照可能不包含任何上述六種行為。這個設計避免模型對所有貼文都強行分類。

v1.0 的 F1 是 0.25，老實說不高。初版最大的問題是多語言數據混在一起訓練，導致中文貼文可能輸出非中文的行為標籤。修正方案是加入語言前綴：`<zh> sentence1` → 只輸出中文行為，`<en> sentence1` → 只輸出英文行為。

## 搜尋架構：四步串起來

TextAction 不是獨立運作的，它和 GeneralTag 深度整合，共同驅動搜尋和推薦。整個搜尋流程分四步：

**Step 1：資料準備**

所有貼文先跑 TextAction 和 GeneralTag 的 inference，中文部分只保留「分享」「推薦」「開箱」「教學」四種有商業價值的行為（「宣傳」太中性、「表達」太感性，對品牌搜尋幫助有限）。然後建立 level2-action mapping 和 level1-action mapping — 這是 TextAction 和 GeneralTag 的聯合索引，把行為和主題標籤對應起來。最後把 action 建入 BM25 的 Elasticsearch index。

**Step 2：訓練 Word Embedding**

用每個 KOL 的 action sequence 作為語料，訓練 FastText skipgram 模型。一個 KOL 的所有 action 排成一列，就像一個句子：`<user1 action1> <user1 action2> <user1 action3>`。這樣訓練出來的 word embedding 能捕捉 action 之間的共現關係 — 經常做「開箱」的 KOL 也常做「推薦」，這兩個 action 的向量就會很接近。

**Step 3：搜尋流程**

使用者輸入關鍵字後，先用 BM25 在 action index 裡找最近的 action（a1），然後從 a1 的 level2/level1 mapping 各取 20 個 candidate，用 Word2Vec 的 cosine similarity 重新排序，取 Top 12（去掉前 2 個，通常和原始 action 太相似沒有擴展價值）。

**Step 4：多關鍵字搜尋**

如果使用者輸入多個關鍵字，每個關鍵字各自走一遍 Step 3，得到各自的 action results，最後合併、去重、按 cosine similarity 排序。

## 搜尋結果品質


看幾個實際的搜尋結果（2023Q3 zh-hant corpus）：

搜「運動」→ 分享健身經驗（0.13）、推薦飲食產品（0.11）、分享減肥經驗（0.11）。這裡可以看到行為維度的價值：不只是找到「運動」相關的 KOL，還區分出了「分享健身經驗」和「推薦飲食產品」兩種不同的行為模式。品牌可以根據自己的需求選擇。

搜「健康」→ 分享醫療經驗（0.15）、推薦健康食品（0.15）、分享讀者經驗（0.14）。結果合理，行為類型多元。

搜「煮飯」→ 分享咖啡廳經驗（0.16）、分享餐廳經驗（0.15）、推薦中秋禮盒（0.14）。這個結果就有點偏了 — 「煮飯」的意圖應該更偏向「教學」類的烹飪內容，但結果偏向「分享」類的飲食體驗。

搜「瑜伽」→ 推薦美甲服務（0.17）、推薦髮型服務（0.17）、推薦上衣產品（0.13）。這個明顯有問題，和瑜伽幾乎沒有關聯。原因是 action embedding 的訓練語料裡瑜伽相關的 action 太少，模型在這塊的向量空間根本沒有區分度——少見的 query 就會被「推」到最近的熱門向量去。

## 在產品中的位置


TextAction 在整個產品架構中連接了四個系統：

搜尋系統 Hermes 用 BM25 + Embedding 做 KOL 排序，TextAction 補上行為維度的搜尋。推薦系統用 ALS/DCR/CHAOS 做品牌-KOL 推薦，TextAction 補上行為相似 KOL 的推薦維度。GeneralTag 提供 Level 1 + Level 2 的主題分類，和 TextAction 的 level-action mapping 構成聯合索引。消費者洞察用貼文情緒分析做基礎，TextAction 把消費者的行為模式也納入分析。

## v1.1 的方向

v1.1 規劃了三個方向：行為萃取模型本身的優化、搜尋品質的系統性驗證、以及和 GeneralTag 結合的 Action Search Understanding v2。

v2 的核心想法是整合 BM25（詞彙匹配）和 embedding（語意匹配），讓 action search 也走混合搜尋的路線。目前 action search 只用 BM25 做初步召回再用 Word2Vec 排序，如果把 action 本身也做 embedding 然後混合搜尋，「瑜伽」這種在 BM25 action index 裡覆蓋率低的 query 可能會有更好的結果。

---

六大行為的分類是靜態的，但社群媒體的內容形式一直在變。短影音讓「開箱」從圖文變成 15 秒影片，直播帶貨讓「推薦」和「教學」的邊界更模糊——一個 KOL 在直播裡一邊教化妝一邊推薦產品，這算教學還是推薦？目前的六分法處理不了這種混合行為，但要怎麼改也不是加幾個類別就能解決的。
