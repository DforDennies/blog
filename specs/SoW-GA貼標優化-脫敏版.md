1. # **概述** {#概述}

   1. ## **專案名稱** {#專案名稱}

   本專案名稱為「GA 自定義貼標優化專案」（以下簡稱本專案）。

   2. ## **專案摘要** {#專案摘要}

   本工作說明書由 Client A（金融業）（以下簡稱甲方）委託愛卡拉互動媒體股份有限公司（以下簡稱本公司或 iKala）執行本案，旨在以甲方全體數位訪客被貼上的 Google 意圖標籤，透過 Vertex AI 的 ML 預測性分析，提升後續導入 LLM 時的標籤推薦精準度。

   與甲方相關人員共同致力於本系統之規劃與建置工作，包括功能性需求以及非功能性需求，如整合功能需求、功能測試與上線輔導等，並提此工作說明書做為專案執行之參考依據。

   

   本專案工作說明書基於：

* 透過事前會議與多次會議討論，得知甲方資訊實際需求。  
* 過往建置經驗與對客戶關係管理專業知識。  
* 對成為甲方資訊長期策略夥伴的誠意與決心。

  3. ## **專案範圍** {#專案範圍}

  專案範圍以工作說明書中專案範圍章節所列項目為限，若有其他關係企業或子公司導入需求，將另案處理，不在本專案範圍之內。

     	若甲方因任何原因要求本公司專案人員赴其他地區或國家工作，則甲方必須依本公司的人事制度與規範，支付所衍生的交通費、機票費與食宿費。若因此改變而影響原訂之專案時程，則按本工作說明書「專案變更管理」章節處理之。


     1. ### 專案將快速驗證技術可行性，包含兩部分：

1. 建立「意圖標籤與產品線關聯性矩陣」：將每位訪客被貼上的 GA 意圖標籤 (In-Market Categories) 及觸發的 GA 自定義事件，以 Vertex AI 的 AutoML 功能計算意圖標籤與產品線的關聯性分數，在 BigQuery 中建立關聯性矩陣。  
2. 建立 GA 目標對象：從 BigQuery 中的關聯性矩陣取出各產品線關聯性分數前幾名的意圖標籤，透過 GA API 在 GA360 中建立目標對象。

   2. ### 因甲方無法直接提供 BigQuery 資料存取權限，故專案將使用甲方提供的 Sample Data，由 iKala 生成測試資料，驗證兩部分流程的可行性。

      3. ### 本專案使用的資料為 iKala 生成的假資料(Mock Data)，ML 模型計算的關聯性分數僅為參考值，故驗收標準以成功運行流程並產出對應標的物即可，不聚焦在提升 ML 模型的精準度。

      4. ### 本專案全程使用 iKala 的 GCP 環境進行開發，iKala 預計使用 BigQuery、Vertex AI 的 AutoML、Cloud Run 執行本專案。

2. # **專案規劃** {#專案規劃}

   1. ## **流程 1-0** {#流程-1-0}

      1. ### 因甲方無法提供 BigQuery 中的真實資料，故由 iKala 製作對應的假資料 (Mock Data)，以驗證流程可行性。

   2. ## **流程 1-1** {#流程-1-1}

      1. ### 建立目標對象設定檔

* ### 以 Google Ads API 的 In-Market Categories (意圖標籤) 檔案隨機抽選 50 個意圖標籤建立表 1，存到 iKala BigQuery 中。   表 1\. 目標對象檔   *(表格圖示 — 已移除)*

  2. ### 建立 5,000 個 user\_pseudo\_id，每人隨機分配 1\~5 個目標對象，做成巢狀格式，模仿在 GA4-BigQuery 內看到的樣子，建立表 2，存到 iKala BigQuery 中。

     ### 表 2\. user\_pseudo\_id 與目標對象分類表      *(表格圖示 — 已移除)*

     3. ### 對這 5,000 個 user 每人隨機製作 3\~10 個事件，每個事件隨機 assign 一個產品線 (從信用卡、信貸、房貸、外幣中隨機挑選一個)，模仿 GA4-BQ 內看到的樣子，建立表 3，存到 iKala BigQuery 中。

     ### 表 3\. user\_pseudo\_id 與 GA事件對照表         *(表格圖示 — 已移除)*

     4. ### 計算意圖標籤與產品線的關聯性分數

* Mapping 表1 \~表 3 建立表 4。  
  表 4\. user\_pseudo\_id、產品線、意圖標籤關聯表  
  *(表格圖示 — 已移除)*  
* 在 Vertex AI 中的 AutoML 嘗試多個關聯度分析模型及關聯性分數計算方式，計算 50 個意圖標籤 與 4 個產品線 (信用卡、信貸、房貸、外幣) 間的關聯性分數 (分數介於 0\~1 之間)。  
* 註：本專案僅建立信用卡、信貸、房貸、外幣等四個產品線的關聯性矩陣。

  5. ### 建立意圖標籤與產品線關聯性矩陣

* 將前一步驟計算完的資料存入 iKala BigQuery 中，建立表 5，首列為產品線名稱、首行為意圖標籤，表格的值為介於 0\~1 的關聯性分數。

  表 5\. 意圖標籤與產品線關聯性矩陣

  *(表格圖示 — 已移除)*


  3. ## **流程 1-2** {#流程-1-2}

     1. 從 BQ 的意圖標籤與產品線關聯性矩陣取出各產品線關聯性前 10 名 (數量待定) 的意圖標籤。  
     2. 將取出的標籤，透過 GA API 在 iKala GA4 建立目標對象。

     註：本專案將使用 Cloud Run 執行流程 1-1 中計算意圖標籤與產品線的關聯性分數、建立意圖標籤與產品線關聯性矩陣，以及流程 1-2，驗證技術可行性。

  4. ## **預計使用的 GCP 元件及環境** {#預計使用的-gcp-元件及環境}

  本專案僅使用 iKala 的 GCP 環境，使用的 GCP 元件包含 BigQuery、Vertex AI 及 Cloud Run。

1. 項目交付後並經甲方驗收完成後，即確認 iKala 對於該等項目之相關應履行義務皆已完成，且該等項目之責任歸屬於驗收完成後即轉移至甲方。
